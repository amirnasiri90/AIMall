# رمز دیتابیس با کاراکتر خاص در DATABASE_URL

اگر رمز PostgreSQL شما کاراکترهای خاص دارد (مثل `)`, `(`, `&`, `|`, `$`, `{`, `}`, `~`, `#`, `?`, `=`, `%`)، در فایل `backend/.env` داخل `DATABASE_URL` باید **رمز را درصد-کد (URL-encode)** کنید وگرنه Prisma خطای **P1013: invalid port number** می‌دهد.

## کاراکترهای رایج و معادل کد

| کاراکتر | کد |
|---------|-----|
| `)` | %29 |
| `(` | %28 |
| `{` | %7B |
| `}` | %7D |
| `&` | %26 |
| `\|` | %7C |
| `~` | %7E |
| `$` | %24 |
| `>` | %3E |
| `<` | %3C |
| `#` | %23 |
| `?` | %3F |
| `=` | %3D |
| `%` | %25 |
| فاصله | %20 |

## مثال

رمز: `eZG){M&~6d>Zc3||0z8&$)(`

معادل کدشده برای استفاده در URL:
```
eZG%29%7BM%26%7E6d%3EZc3%7C%7C0z8%26%24%29%28
```

پس در `.env`:
```
DATABASE_URL=postgresql://aimall_user:eZG%29%7BM%26%7E6d%3EZc3%7C%7C0z8%26%24%29%28@127.0.0.1:5432/aimall?schema=public
```

## راه دیگر

اگر ترجیح می‌دهید رمز را عوض کنید، یک رمز **بدون** کاراکتر خاص انتخاب کنید (فقط حروف و اعداد)، مثلاً:
`AimallSecurePass2024!`

در آن صورت نیازی به encode در URL نیست.
